The four security layer cards in a `.security-grid`:

**Card 1 — Landlock LSM** (icon: shield SVG, color: green #22c55e)
- Title: "Landlock LSM"
- Description: "Linux Security Module restricting filesystem access at the kernel level. The sandbox process cannot access paths outside its granted set &mdash; even if the agent finds a code execution vulnerability."
- Frame from EG perspective (mention Evidence Gate validates this is configured).

**Card 2 — seccomp** (icon: filter/funnel SVG, color: blue #2563eb)
- Title: "seccomp Filtering"
- Description: "System call filter that limits which kernel operations the sandbox process can invoke. Blocks dangerous syscalls like <code>ptrace</code>, <code>mount</code>, and <code>reboot</code> before they reach the kernel."

**Card 3 — Network Namespace** (icon: network/globe SVG, color: purple #7c3aed)
- Title: "Network Namespace Isolation"
- Description: "Each sandbox runs in its own network namespace with deny-by-default egress policy. Only endpoints explicitly approved in the blueprint can be reached. Unapproved requests are blocked and surfaced for operator approval."

**Card 4 — Inference Control** (icon: route/cpu SVG, color: slate #475569)
- Title: "Inference Control"
- Description: "LLM inference requests route through the OpenShell gateway &mdash; never directly from the agent process. The gateway enforces model allowlists, rate limits, and cost caps before forwarding to the provider."

Below the security-grid, add the `.security-constraints` div with two constraint summary boxes:

**Box 1 — Filesystem Constraints** (border-left-color: #22c55e)
- Heading: "Filesystem"
- List items:
  - "<code>/sandbox</code> &mdash; agent working directory (read + write)"
  - "<code>/tmp</code> &mdash; temporary files (read + write)"
  - "All other paths &mdash; read-only or inaccessible"
  - "System binaries, configs, and host mounts are never writable"

**Box 2 — Network Constraints** (border-left-color: #7c3aed)
- Heading: "Network"
- List items:

  - "Approved endpoints listed in <code>blueprint.yaml</code>"
  - "Unapproved requests blocked and queued for operator review"
  - "Inference traffic routed through gateway, not direct from agent"
