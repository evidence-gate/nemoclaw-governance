**Step 3: Workflow YAML content inside the code-block (rendered with syntax highlight spans):**

The workflow YAML shows three parallel jobs:
1. `validate-blueprint` job: Uses `evidence-gate/evidence-gate-action@v1` with gate_type "nemoclaw_blueprint" to validate blueprint.yaml
2. `validate-policy` job: Uses `evidence-gate/evidence-gate-action@v1` with gate_type "nemoclaw_policy" to validate policy.yaml
3. `enforce-budget` job: Uses `evidence-gate/evidence-gate-action@v1` with gate_type "custom" and evidence_files referencing agentgov.config.json

Full YAML content:
```
name: NemoClaw Governance
on: [pull_request]

permissions:
  contents: read
  checks: write

jobs:
  validate-blueprint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      # Validate NemoClaw blueprint.yaml
      - name: Blueprint Gate
        uses: evidence-gate/evidence-gate-action@v1
        with:
          gate_type: "nemoclaw_blueprint"
          evidence_files: "blueprint.yaml"

  validate-policy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      # Validate sandbox policy constraints
      - name: Policy Gate
        uses: evidence-gate/evidence-gate-action@v1
        with:
          gate_type: "nemoclaw_policy"
          evidence_files: "policy.yaml"

  enforce-budget:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      # Validate agentgov budget configuration
      - name: Budget Gate
        uses: evidence-gate/evidence-gate-action@v1
        with:
          gate_type: "custom"
          phase_id: "budget"
          evidence_files: "agentgov.config.json"
```
