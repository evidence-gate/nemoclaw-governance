NemoClaw official documentation facts to implement:

Sandbox Lifecycle (5 stages):
1. Resolve — Plugin resolves blueprint version and downloads artifact
2. Verify — Plugin checks blueprint signature and integrity
3. Plan — Blueprint determines OpenShell resources needed
4. Apply — Blueprint invokes OpenShell CLI to create and configure resources
5. Status — Blueprint reports sandbox readiness and endpoint information

Inference Routing (3 profiles):
1. NVIDIA Cloud — Nemotron 3 Super 120B via build.nvidia.com (production, highest capability)
2. Local NIM — NIM container on local network (testing, air-gapped environments)
3. Local vLLM — vLLM server on localhost (offline dev, fast iteration)
- Providers can be switched at runtime without sandbox restart
